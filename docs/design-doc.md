## 1. 产品概述

### 1.1 一句话描述

**将占星学（行星位置、相位、宫位系统）与股票市场数据结合，为用户提供基于天象的投资决策参考。**

### 1.2 产品定位

- **目标用户**：对占星感兴趣的投资者 / 寻求另类分析视角的交易者
- **核心价值**：提供一个传统技术分析之外的"另类数据"视角
- **差异化**：不是单纯的占星软件，也不是单纯的股票软件，而是二者的交叉融合

### 1.3  slogan

> "当天象指引投资方向" — *When the stars align with your portfolio.*

## 2. 核心假设与理念

### 2.1 占星-市场映射假设

| 占星元素 | 对应市场含义 |
|----------|-------------|
| 太阳   | 市场整体情绪、大盘趋势 |
| 月亮   | 短期波动、散户情绪 |
| 水星   | 信息流、新闻、通信板块 |
| 金星   | 消费、奢侈品、金融 |
| 火星   | 国防、工业、能源（行动力板块）|
| 木星   | 扩张、增长型股票、牛市 |
| 土星   | 紧缩、防御型、熊市压力 |
| 天王星 | 科技创新、突破性股票 |
| 海王星 | 泡沫、欺诈风险、大宗商品 |
| 冥王星 | 深度变革、重组、加密货币 |

### 2.2 相位影响力

| 相位 | 影响类型 | 置信度权重 |
|------|---------|-----------|
| 合相 (0°)   | 强烈集中 | ×1.5 |
| 三分相 (120°) | 和谐助力 | ×1.3 |
| 六分相 (60°)  | 机会辅助 | ×1.2 |
| 四分相 (90°)  | 张力挑战 | ×0.8 |
| 对分相 (180°) | 对立博弈 | ×0.7 |

### 2.3 关键原则

1. **不承诺 100% 准确** — 占星是概率工具，不是预言
2. **结合传统分析** — 占星信号需与 TA/FA 交叉验证
3. **个人星盘校准** — 用户输入生时信息后，系统按用户本命盘个性化推荐

## 3. 用户角色

| 角色 | 描述 | 核心需求 |
|------|------|---------|
| **新手占星投资者** | 懂一点星座，想尝试用占星炒股 | 简单易懂的天象解读、买入/卖出信号 |

## 8. 算法逻辑

### 8.1 占星评分算法 (AstroScore 0-100)

```
Score = (PlanetaryWeight × 40) + (AspectScore × 30) + (TransitMatch × 20) + (PersonalFit × 10)
```

**权重说明**：
- **PlanetaryWeight (40%)**
  - 计算行星与个股所在板块的对应关系
  - 例：火星在狮子座 → 科技/AI板块 +18pt
- **AspectScore (30%)**
  - 当前形成的有利相位加分，挑战相位减分
  - 三分相 +5pt/个 · 四分相 -3pt/个
- **TransitMatch (20%)**
  - 过运行星是否触发个股的"出生星盘"关键点
- **PersonalFit (10%)**
  - 个股板块是否与用户的个人星盘匹配

### 8.2 预测方向判定

```
if (Score >= 75) → 强烈看涨 (🔵)
if (Score >= 60) → 温和看涨 (🟢)
if (Score >= 40) → 中性观望 (🟡)
if (Score >= 25) → 温和看跌 (🟠)
else → 强烈看跌 (🔴)
```

### 8.3 历史准确率追踪

```
Accuracy = 正确预测次数 / 总预测次数

按时间段追踪：
- 7日滚动准确率
- 30日滚动准确率
- 所有时间准确率

细分维度：
- 按股票
- 按板块
- 按行星相位类型
```

## 9. 技术架构建议

### 9.1 推荐技术栈

| 层级 | 技术 | 理由 |
|------|------|------|
| **前端** | React / Vue 3 + TailwindCSS | 原型已是 Vue 风格，可无缝迁移 |
| **后端 API** | Python (FastAPI) 或 Node.js (Express) | Python 有丰富天文学库 |
| **天文计算** | Swiss Ephemeris / PyEphemeris | 专业占星计算引擎，精度达到角秒 |
| **市场数据** | Yahoo Finance API / Alpha Vantage | 免费实时/历史股票数据 |
| **数据库** | PostgreSQL + TimescaleDB | 时序数据优化 |
| **AI 分析** | OpenAI GPT-4 API 或本地 LLM | 自然语言解读天象影响 |
| **部署** | Docker + Vercel / Railway | 快速原型部署 |

### 9.2 数据流

```
[天文学引擎] ─→ 行星位置数据 ─→ [评分引擎] ─→ AstroScore
                         ↑                      ↓
[市场数据 API] ─→ 股票数据 ────────→ [预测引擎
